### Задание 1. Состояния и обработка нажатия

В первом задании был создан прямоугольник `Rectangle`. Для него задано состояние `clicked`, в котором меняются координаты элемента по осям `x` и `y`.

```qml
Rectangle {
    id: rect
    width: 400
    height: 100
    color: "lightpink"
    states: [
        State {
            name: "clicked"
            PropertyChanges {
                target: rect
                x: 300
                y: 1000
            }
        }
    ]
}
```

Для обработки нажатия используется `MouseArea`, которая занимает всю область прямоугольника:

```qml
MouseArea {
    id: mouseArea
    anchors.fill: parent
    onClicked: rect.state === "clicked" ? rect.state = "" : rect.state = "clicked"
}
```

При нажатии на прямоугольник состояние переключается: если элемент уже находится в состоянии `clicked`, состояние сбрасывается, иначе устанавливается `clicked`. В результате прямоугольник меняет свое положение на странице.

Результат выполнения первого задания: изучено создание состояний QML-элементов и переключение состояния по событию нажатия.

### Задание 2. Анимация элементов

Во втором задании были созданы три прямоугольника, размещенные вертикально внутри контейнера `Column`. Для каждого прямоугольника задан свой цвет, размер и анимация вращения.

```qml
Column {
    anchors.centerIn: parent
    spacing: 300

    Rectangle {
        id: rect1
        color: "lightpink"
        width: 200
        height: 100

        RotationAnimation on rotation {
            loops: Animation.Infinite
            from: 0
            to: 90
        }
    }
}
```

Для второго и третьего прямоугольника используется такая же анимация, но с другими конечными значениями угла поворота:

```qml
RotationAnimation on rotation {
    loops: Animation.Infinite
    from: 0
    to: 180
}

RotationAnimation on rotation {
    loops: Animation.Infinite
    from: 0
    to: 360
}
```

Свойство `loops: Animation.Infinite` делает анимацию бесконечной. Свойства `from` и `to` задают начальный и конечный угол вращения.

Результат выполнения второго задания: были созданы несколько анимированных элементов и изучена работа `RotationAnimation` в QML.

### Задание 3. Загрузка XML и вывод списка

В третьем задании была реализована загрузка XML-данных с сайта Центрального банка России. Для этого используется модель `XmlListModel`.

```qml
XmlListModel {
    id: model
    source: "https://cbr.ru/scripts/XML_val.asp?d=O"
    query: "/Valuta/Item"

    XmlRole { name: "name"; query: "Name/string()" }
    XmlRole { name: "nominal"; query: "Nominal/string()" }
    XmlRole { name: "parentCode"; query: "ParentCode/string()" }
}
```

Источник данных задается через свойство `source`. Запрос `query: "/Valuta/Item"` выбирает элементы XML-документа, которые будут использоваться как записи модели. С помощью `XmlRole` из каждой записи извлекаются поля:

- `name` — название валюты;
- `nominal` — номинал;
- `parentCode` — родительский код.

Также добавлен обработчик изменения статуса модели:

```qml
onStatusChanged: {
    if (status === XmlListModel.Ready) {
        console.log("Got", count, "Market Lib")
    } else if (status == XmlListModel.Error) {
        console.log("Error loading Market Libs:", errorString())
    }
}
```

Если данные успешно загружены, в консоль выводится количество полученных записей. Если во время загрузки произошла ошибка, в консоль выводится текст ошибки.

Для отображения данных на экране используется `ListView`:

```qml
ListView {
    anchors.fill: parent
    model: model
    spacing: 100
    delegate: Item {
        width: parent.width
        height: 100
        Column {
            Text {
                text: "Name: " + name
            }
            Text {
                text: "Nominal: " + nominal
            }
            Text {
                text: "ParentCode: " + parentCode
            }
        }
    }
}
```

В качестве модели списка используется `XmlListModel`. Для каждой записи создается делегат, внутри которого через `Text` выводятся название валюты, номинал и родительский код.

Результат выполнения третьего задания: приложение получает XML-данные из внешнего источника и отображает их в виде списка на главной странице.
