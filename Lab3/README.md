## Цель работы

Изучить работу со стеком страниц в приложениях Aurora OS, загрузку отдельных QML-компонентов через `Loader`, а также создание выпадающего меню `PullDownMenu` и взаимодействие элементов интерфейса через обработчики событий.

## Задания по методичке

В рамках лабораторной работы необходимо было выполнить три задания:

1. Реализовать переход между страницами с помощью `pageStack.push()` и возврат назад с помощью `pageStack.pop()`.
2. Создать отдельный QML-компонент и загрузить его на страницу через `Loader`.
3. Создать выпадающее меню `PullDownMenu`, добавить в него пункты действий и связать их со значением элемента `Slider`.

### Задание 1. Навигация между страницами

В первом задании была реализована навигация по стеку страниц. Для размещения кнопок использовался контейнер `Column`, расположенный по центру страницы.

```qml
Column {
    width: parent.width
    spacing: Theme.paddingLarge
    anchors.centerIn: parent

    Button {
        text: "Вперед"
        anchors.horizontalCenter: parent.horizontalCenter
        onClicked: pageStack.push(Qt.resolvedUrl("MainPage.qml"))
        backgroundColor: "lightpink"
    }

    Button {
        text: "Назад"
        anchors.horizontalCenter: parent.horizontalCenter
        onClicked: pageStack.pop()
        backgroundColor: "lightyellow"
    }
}
```

Кнопка `Вперед` вызывает метод `pageStack.push()` и добавляет новую страницу `MainPage.qml` в стек. Кнопка `Назад` вызывает метод `pageStack.pop()` и возвращает пользователя на предыдущую страницу.

За счет свойства `pageStack.depth` в заголовке можно видеть, сколько страниц находится в стеке. При переходе вперед значение увеличивается, при возврате назад уменьшается.

Результат выполнения первого задания: реализована базовая навигация между страницами приложения Aurora OS.

### Задание 2. Загрузка отдельного QML-компонента

Во втором задании был создан отдельный QML-компонент `RectComponent.qml`. Он расположен в папке `qml/pages` и описывает прямоугольник с заданными параметрами:

```qml
import QtQuick 2.0

Rectangle {
    color: "#36FF00"
    width: 400
    height: 150
}
```

Для подключения компонента к главной странице используется `Loader`:

```qml
Loader {
    id: rect
    anchors.centerIn: parent
    source: "RectComponent.qml"
}
```

Свойство `source` указывает путь к компоненту, который необходимо загрузить. Благодаря `anchors.centerIn: parent` загруженный прямоугольник располагается по центру страницы.

Файл компонента также добавлен в проект в `com.byprogger.Lab3.pro`:

```pro
DISTFILES += \
    qml/pages/RectComponent.qml \
```

Результат выполнения второго задания: создан отдельный QML-компонент и выполнена его загрузка на страницу через `Loader`.

### Задание 3. Выпадающее меню и слайдер

В третьем задании была реализована страница с прокручиваемой областью `SilicaFlickable`, выпадающим меню `PullDownMenu` и слайдером `Slider`.

```qml
SilicaFlickable {
    anchors.fill: parent

    PullDownMenu {
        MenuLabel { text: "Информационная ветка" }

        MenuItem {
            text: "Действие 1"
            onClicked: {
                console.log("Действие 1")
                slider.value = 20
            }
        }
    }

    Slider {
        id: slider
        width: parent.width
        maximumValue: 100
        minimumValue: 0
        value: 0
    }
}
```

В меню добавлены три пункта:

```qml
MenuItem {
    text: "Действие 1"
    onClicked: {
        console.log("Действие 1")
        slider.value = 20
    }
}

MenuItem {
    text: "Действие 2"
    onClicked: {
        console.log("Действие 2")
        slider.value = 40
    }
}

MenuItem {
    text: "Действие 3"
    onClicked: {
        console.log("Действие 3")
        slider.value = 60
    }
}
```

Каждый пункт меню имеет обработчик `onClicked`. При выборе пункта в консоль выводится название действия, а значение `slider.value` изменяется на соответствующее число: `20`, `40` или `60`.

Слайдер настроен на диапазон от `0` до `100`:

```qml
Slider {
    id: slider
    width: parent.width
    maximumValue: 100
    minimumValue: 0
    value: 0
}
```

Результат выполнения третьего задания: создано выпадающее меню, пункты которого управляют значением слайдера и выводят сообщения в консоль.
